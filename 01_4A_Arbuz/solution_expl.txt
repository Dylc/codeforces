так что каждый получит целое числоб а если нет - то нельзя
Есть одно исключение - 2 - целое число не делящееся поровну

3. Bit solution
Исрользуются битвайзовые операции
x&y - даст 1 только на 1 и 1 (AND)
x|y - даст 1 на все кроме 0 и 0  (OR)

Сначала булеанб мы делаем блок a<3 (почему такой порядок?)
если правда - то получаем 1, ложь - 0
То есть если число более чем 2 то будет 1

Далее a&1
Что дает нам это?
Каждое четное сичло заканичвается в двоичном формате на 0 (а нечетное на 1)
напрмер 
0 - 0       // четное
1 - 1
2 - 10      // четное
3 - 11
4 - 100     // четное
5 - 101
6 - 110     // четное
7 - 111
8 - 1000    // четное

То есть делая "a&1" - мы делаем И оператор на последнюю цифру числа 
- если было 0 - то получим всегда 0 - т.е число четное
- если было 1 - то получим всегда 1 - т.е число нечетное


Далее сравниваем наше число с рузультатом блока a<3
Оно изменится только если число было менее 3
например для 2
2 = 10
2<3     ==> 1
10&1    ==> 0
0|1     ==> 1 - print "NO"


например для 4
4 = 100
4<3     ==> 0
100&1   ==> 0
0|0     ==> 0 - print "YES"


например для 5
5 = 101
1<3     ==> 0
101&1   ==> 1
1|0     ==> 1 - print "NO"
